# DS 422 — Final Project Rubric  
**GIS in R • Fall 2025**  
**Total: 34 points**

**Due Date:** December 2 @ 2:30 PM  
**Presentation:** December 2 @ 2:30 PM  
**Group Size:** Up to 3 students  
**Deliverables:** Public GitHub repository + R code + 10-minute presentation

---

## 1. GitHub Project, Collaboration & Workflow — *16 points*

### **A. Team Contributions (8 pts)**
- All team members push meaningful code to GitHub; commit history reflects genuine collaboration.  
- Shared workload and full participation in work sessions (attendance counted unless excused).

### **B. Repository Quality (6 pts)**
- Public repository with clear structure, organized folders, and reproducible R code.  
- README includes:  
  - Project summary  
  - Data description or link  
  - **Three-sentence bios for each team member, with photo and link to GitHub/website/LinkedIn**

### **C. Research Question & Data Setup (2 pts)**
- Well-defined research question.  
- Appropriate dataset(s) included in the repo or linked in the README.

---

## 2. Analysis & Code Application — *8 points*

- Effective application of GIS in R (spatial joins, mapping, tidy workflows, etc.).  
- Code is clean, readable, and well-commented.  
- Analysis directly addresses the research question.

---

## 3. Visualizations & Results — *6 points*

- Final maps/figures are clear, labeled, and visually polished.  
- Results are correctly interpreted and tied back to the research question.

---

## 4. Final Presentation — *4 points*

**10 minutes + 1–2 minute GitHub walkthrough**

- Background and research question clearly stated.  
- Summary of data and analysis.  
- Findings and visualizations presented effectively.  
- Brief walkthrough of GitHub repository (structure, scripts, README) and a clear conclusion.
